- **`components/shared/data-as-of.tsx`** — the standard label, one vocabulary:
  - `facts` mode: "Data as of Tue 1 Jul" (daily/lifetime facts)

    components (no hydration drift).

### REAL BUG FIXED — mailboxes tab crashed in production for cylindo

`mailbox-inventory-table.tsx` grouped by lifecycle but had NO group for
**resting** or **parked**, and its fallback pushed to a non-existent
`result["other"]` → `TypeError` → whole tab = error screen. Cylindo has
~50 resting boxes at any time (1-week-active/1-week-rest rotation), so
the tab was down for exactly the client with the most mailboxes. Added
Resting group (violet), an "Other" **catch-all group** so unknown
lifecycles can never crash the tab again, and resting/parked filter
options. (1 cylindo box currently lands in "Other" — odd lifecycle value,
now visible instead of fatal.)

## 2. Env notes added this session

- The preview browser's screenshot tool times out on pages with the
  TodayLiveStrip (infinite `animate-ping` defeats visual-stability wait) —
  use `preview_snapshot` / `preview_inspect` / eval instead.
- `.next/dev/types/validator.ts` can be left CORRUPTED (torn write) when
  the dev server is killed — breaks `tsc` AND `next build` with nonsense
  errors (e.g. "Cannot find name 'dler'"). Fix: `rm -rf .next/dev`.
- Local force-dynamic SSR renders can take 10-30s (the documented node
  DNS flakiness) — pages show skeletons meanwhile; not an app bug.
- Smoke user password was rotated this session (in-session only, not
  written down — reset again via admin API when needed). Smoke user
  still owed deletion after Omar's review.
